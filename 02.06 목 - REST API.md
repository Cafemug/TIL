## REST API 디자인 가이드

- URI는 정보의 자원을 표현해야 한다.
- 자원에 대한 행위는 HTTP Method(GET, POST, PUT, DELETE)로 표현한다.


### URI 설계시 주의사항
- 슬래시 구분자(/)는 계층 관계를 나타내는데 표현

- URL 마지막문자로 슬래시(/)를 포함하지 않는다

- 하이픈(-)은 URI 가독성을 높이는데 사용(kebab-case)

- 밑줄(_)은 URI에 사용하지 않는다

- URI 경로에는 소문자가 적합하다

- 파일 확장자는 URI에 포함시키지 않는다
